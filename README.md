# Andersson (2019) Replication

This folder contains a course replication of Andersson (2019), focused on Sweden's carbon tax and transport-sector CO2 emissions.

## Project layout

- `code/`: standalone script and shared helper functions.
- `inputs/data/`: cleaned CSV inputs used by the replication.
- `inputs/original-code/`: original paper replication materials kept for reference.
- `writeup/`: report source plus generated markdown, PDF, figures, and cache.
- `outputs/standalone/`: figure PDFs generated by the standalone script.
- `papers/`: paper PDFs and related reference files.

## Recommended workflow

1. Edit `writeup/writeup.Rmd` for the report narrative and embedded analysis.
2. Edit `code/replication_helpers.R` only for logic that should stay identical across the report and the standalone script.
3. Use `code/andersson_replication.R` when you want a quick end-to-end script run without rebuilding the full writeup.

## Rebuild outputs

Run the standalone script:

```r
Rscript code/andersson_replication.R
```

Rebuild the markdown and purl outputs:

```r
knitr::knit("writeup/writeup.Rmd", output = "writeup/writeup.md")
knitr::purl("writeup/writeup.Rmd", output = "writeup/writeup_purl.R", documentation = 0)
```

Render the PDF report:

```r
rmarkdown::render("writeup/writeup.Rmd")
```

## Notes

- `writeup/writeup.md`, `writeup/writeup.pdf`, `writeup/writeup_purl.R`, `writeup/figure/`, `writeup/cache/`, and `outputs/standalone/` are generated outputs and should be refreshed after source changes.
- `code/replication_helpers.R` keeps labels, treatment years, plotting styles, and repeated analysis steps synchronized across the project.
